 RG1      Sincronización de Listas con catálogo                                                                                                                                                                                         Todos los campos de tipo selección (lista desplegable) deben poblarse exclusivamente con los datos gestionados en la pestaña de Administración. No usar contenido estático.         ok
 RG2                Validación de Multivalores                                                                                                                                                Cuando para el campo se permite registrar multiples valores, se debe controlar:\n- que no seleccione el mismo valor mas de una vez. \n- solo es obligatorio registrar un valor, un segundo o tercer valor es opcional         ok
 RG3                        Filtrado en Cascada                                                          Cuando los campos están relacionados, al seleccionar en la primera lista se debe filtrar los valores a desplegar en la segunda lista. Ejemplo: al seleccionar una "Problemática", la lista de "Problema" asociada debe filtrarse automáticamente para mostrar solo los ítems relacionados a esa categoría.         ok
 RG4         Administración Listas desplegables                                                     \t• No permite eliminar un valor de la lista si éste ya se encuentra incluido en algún registro de llamada - ver como se haria\n\t• Permitir Activar e Inactivar valores - ok\n\t• En las listas desplegables solo mostrar al usuario los valores activos- ok\n\t• Permite cambiar o ajustar descripciones - ok        NaN
 RG5                Campos Obligatorios llamada                                                                                                                                                                  Campos que están disponibles en todas las Problemáticas y su diligenciamiento es obligatorio:\nL_Orientador\nL_Fecha\nL_Hora\nL_Resultado\nL_Duración\nL_Medio_Contacto\nL_Como_Conoce\nL_Sintesis        NaN

 RG7                                     Estado                                                                                                                                                                                                                                         Al desplegar valores en las listas para que el orientador seleccione, solo debe mostrar los valores donde Estado = "Activo"        RG4
 RG8 Codificación para identificador de llamada                                                                                                        Las reglas identificadas para la asignación del identificador de llamada son:\n-Letra correspondiente al año + Consecutivo dentro del año.  \n-Al acabar alfabeto empezaron combinando letras consecutivas, ejemplo: AA+###, AB###. \n-En el 2026 es AK###.         NaN
 RG9                                        Año                                                    La letra que identifica los códigos generados para las llamadas recibidase en el año actual debe ser registrado como parámetro general  para que pueda ser tomado desde las diferentes pantallas donde se requiere usarlo.\nEl 1 de enero de cada año, este valor debe ser actualizado al valor correspondiente.        NaN
RG10              Aseguramiento listas maestras                                                                                                            Para seleccionar Orientador siempre se debe hacer desplegando la lista de valores desde el  maestro de Orientadores.\nAsí mismo, para seleccionar  Profesional siempre se debe hacer desplegando la lista de valores desde el  maestro de Profesionales.        NaN
